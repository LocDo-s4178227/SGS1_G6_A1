================================================================================
                    PROJECT WIREFRAME DOCUMENTATION
                Service Marketplace with User Accounts
================================================================================

PROJECT OVERVIEW:
This project consists of two integrated modules:
1. USER ACCOUNT MANAGEMENT MODULE - Authentication, profiles, account management
2. SHOPPING CART MODULE - E-commerce products, marketplace requests, proposals

Both modules share a unified backend API and styling system.

================================================================================
USER FLOWS
================================================================================

FLOW 1: NEW USER SIGNUP → MARKETPLACE
1. Visit auth.html
2. Fill registration form (email, password, user type)
3. Submit → Stored in database
4. Redirected to profile.html
5. (Optional) Complete profile information
6. Browse products or post requests
7. Navigate to products.html or request-board.html

FLOW 2: EXISTING USER LOGIN → SHOPPING
1. Visit auth.html
2. Enter credentials
3. Validated against database
4. Stored in localStorage (userId, userRole)
5. Redirected to profile.html or products.html
6. Can now add items to cart
7. Proceed to checkout.html
8. Complete order → order-confirmation.html

FLOW 3: PROFESSIONAL RESPONDS TO REQUEST
1. Browse request-board.html
2. Click "View Details" on request-details.html
3. Click "Submit Proposal" → Modal opens
4. Fill proposal form (bid amount, timeline, message)
5. Submit → Stored in database
6. Proposal visible to requester
7. Enter negotiation via messaging-hub.html
8. Back-and-forth price negotiation
9. Accept proposal → Converts to cart item
10. Checkout and complete order

FLOW 4: MANAGE ACCOUNT
1. Login to profile.html
2. Edit personal info (Profile tab)
3. Change password (Security tab)
4. Manage preferences (Account tab)
5. Can deactivate/delete account with confirmation
